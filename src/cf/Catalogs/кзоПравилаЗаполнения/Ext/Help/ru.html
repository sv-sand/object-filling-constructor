Документ состоит из нумерованных групп реквизитов.

1. Шапка - группа основных реквизитов документа влияющих на будущий состав настроек и событий их выполнения.

1.1 Тип объекта - доступно для выбора один любой тип документа(!в будущем планируются справочники!).

При выборе формируется список реквизитов и табличных частей содержащихся в этом объекте(группа 2. Доступные поля).

1.2 Событие - список доступных событий объекта при наступлении которых будет выполнятся попытка заполнения.

1.2.1 Обработка заполнения - при создании нового объекта. При этом не важно было ли основание.

1.2.2 При копировании - при создании нового объекта на основании существующего объекта того же типа.

Эти события взаимоисключают друг-друга. Те при создании нового на основании не вызовется событие при копировании и обратно.

1.2.3 Перед записью - перед тем, как объект будет записан в базу, есть возможность автоматически заполнить или изменить необходимые реквизиты. 

1.2.4 ...возможно расширение списка в будущем...

Все процедуры подписок на события выполняются после окончания работы процедур указанных в самом объкте в конфигураторе.

Те можно представить заполняемость объекта следующим образом:

Обработка заполнения объекта + 1.2.1 обработка заполнения

или   +  первоначальное программное заполнение формы  +  пользовательносе заполнение формы   +   Перед записью програмнное заполнение  + 1.2.3  Перед записью                                                        

При копировании объекта + 1.2.2 при копировании события

1.3 Табличная часть - поле в котором можно указать табличную часть выбранного объекта. При этом изменится поведение попытки заполнения таким образом, что будет проверятся каждая строка ТЧ на удовлетворение условиям указанных в

2. Доступные поля - список реквизитов подчиненных объекту, которые можно использовать для настройки правил заполнения

Так же добавлены специализированные поля:

(*) Это новый - свойство указывающее был ли записан объект (прог. читай ЗначениеЗаполнено(Объект.Ссылка)).

(*) <Имя ТЧ>:Количество строк - количество строк  указанной в <Имя ТЧ> табличной части (прог. читай ТЧ.Количество() ).

3. Правила отбора - список условий отбора по реквизитам заполняемого объекта. Представляет собой таблицу отбора в типичном отчете созданном с помощью СКД.

Состоит из элементов группы 2. Доступные поля, знаков сравнения ,значений ИБ.  

Условия могут быть сгруппированы.

4. Проверка - сервисная группа. Используется, когда пользователь настравивающий группу 3. Правила отбора хочет проверить существующий объект на соответствие настроенным правилам.

Те позволяет ответить на вопрос: "Применено ли было бы текущее правило заполнения для существующего объекта?"

Значения проверочного поля не сохраняются в ИБ.

5. Правила заполнения - состоит из двух таблиц:

5.1 - представляет собой выбранные поля к заполнению и значения их заполнения (!добавить возможность заполнения из значений элементов группы 2. Доступные поля!).

Именно эти значения буду указаны при проверке объекта на соответствия условиях из группы 3. Правила отбора

Строки добавляются перетаскиванием элементов из группы 2. Доступные поля.

Для добавляения доступны реквизиты шапки и табличных частей.

Исключениями являются:

- стандартные реквизиты (Проведено, Номер, Дата и тп)

- специализированные поля из 2. Доступные поля

- реквизиты подчиненные другим реквизитам (например, Договор Контрагента - Организация)

5.2 - таблицу пользователей. для которых это создаваемое правило будет применяться.

Доступно к редактированию, если только указано Истина для контсанты "Используется подсистема БСП "Пользователи"".

При включенном флаге константы будет происходить проверка текущего пользователя(редактирующего объект) 

и пользователей указаннных в этом списке.

Если список не заполнен или флаг константы в значении Ложь, тогда правило применяется ко всем пользователям ИБ.

6. Дополнительно - сервисная группа для указания пользовательских комментариев и ответсвенного пользовавтеля за текущий документ-правило.

Поле "Ответственный" доступно, когда включена контсанта "Используется подсистема БСП "Пользователи"".

По умолчанию указывается текущий пользователь создавший  документ.
